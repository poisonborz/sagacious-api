Sagacious REST API Documentation

For a general overview and goals of the API, see the General API docs

About this documentation

The Sagacious REST API follows those of the GraphQL endpoints. Since underlying data structures are the same, and GraphQL's schemes lend themselves to self-documentation better, the full list of endpoints to query, post to, and their return value in the GraphQL documentation.

This page details the syntax to access same endpoints, and the different methods or property results, if any.

Authorization is not part of this API endpoint. For that, please refer to the General API docs.

Any REST endpoint is only accessible with a valid access token (authorization header).

General

Queries / GET requests - any entity is queryable with GET requests in the usual REST manner, using its usual singular term. All parameters are passed as is. Multiple parameter values, like ids are separated with commas.

https://api.acmecorp.com/rest/asset/?id=gX4a3F8O,7MGdFR4m

https://api.acmecorp.com/rest/group/?id=4DFdF88m&getAllChildren=true

Methods will map the same way as entity calls:

https://api.acmecorp.com/rest/setLastReadLoginInterrupt/?loginInterruptId=F8OdghF4

Mutations / POST requests - for the sake of simplicity, and since methods are explicitly named, all mutating queries are performed using only POST requests - values should be sent as x-www-form-urlencoded, except for file content, which should use multipart/form-data.
